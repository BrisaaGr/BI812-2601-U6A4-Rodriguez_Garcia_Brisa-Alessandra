01 — Contexto histórico

Historia y Protagonistas

Cada plataforma nació de una necesidad específica dentro del ecosistema de datos. Conocer su origen permite comprender sus fortalezas y limitaciones actuales.

Cloudera
Fundada: 27 jun 2008  ·  Santa Clara, CA
Christophe Bisciglia, Amr Awadallah, Jeff Hammerbacher, Mike Olson

Surgió cuando ingenieros provenientes de Google, Yahoo, Facebook y Oracle identificaron la necesidad de comercializar Apache Hadoop para empresas. En 2009 se incorporó Doug Cutting, co-creador de Hadoop. En 2021 fusionó con Hortonworks y fue adquirida por KKR y CD&R. Hoy ofrece la Cloudera Data Platform (CDP) con capacidades híbridas y multinube.

Hadoop / Híbrido
Databricks
Fundada: 2013  ·  San Francisco, CA
Ali Ghodsi, Ion Stoica, Matei Zaharia, Patrick Wendell, Reynold Xin, Andy Konwinski, Arsalan Tavakoli

Nacida del proyecto Apache Spark en el AMPLab de UC Berkeley. Sus fundadores, siete doctores en Ciencias de la Computación, buscaron resolver las limitaciones del modelo open-source de Spark. En 2023 adquirió MosaicML por $1.3 MmD y en 2024 adquirió Tabular. Para enero de 2026 reportó ingresos ARR de $5.4 MmD.

Lakehouse / Apache Spark
AWS Big Data
Amazon Web Services: 2006  ·  Seattle, WA
Jeff Bezos (Amazon)  ·  Andy Jassy (AWS)

Amazon Web Services lanzó S3 y EC2 en 2006, estableciendo la nube pública. Su ecosistema Big Data incluye Amazon EMR (Hadoop/Spark gestionado), Amazon Redshift (data warehouse columnar), AWS Glue (ETL serverless), Amazon Kinesis (streaming) y AWS Lake Formation. Es la plataforma de nube con mayor cuota de mercado global (~33%).

Multi-servicio / Cloud Native
Snowflake
Fundada: 2012  ·  San Mateo, CA
Benoît Dageville, Thierry Cruanes, Marcin Żukowski

Fundada por ex-arquitectos de Oracle con la visión de construir un data warehouse nativo en la nube. Operó en modo stealth hasta 2014. En 2020 realizó el IPO más grande de software en la historia de NYSE ($3.4 MmD). Su arquitectura de separación compute/storage y el modelo de "pay per query" revolucionó el mercado analítico SQL.

Data Cloud / SaaS
Dremio
Fundada: 2015  ·  Santa Clara, CA
Tomer Shiran, Jacques Nadeau, Kelly Stirman

Nacida con la misión de eliminar el ETL y permitir consultas directas sobre datos en object storage. Desarrolló el concepto de "Data Lakehouse" con Apache Arrow y Apache Iceberg como tecnologías centrales. Su motor de aceleración "Data Reflections" permite consultas sub-segundos sin necesidad de mover datos. Levantó $400M en rondas series C, D y E.

Open Lakehouse / No-ETL

Línea de Tiempo del Ecosistema Big Data

2006
Amazon Web Services lanza S3 y EC2, sentando las bases del cómputo en nube.
2008
Cloudera fundada por ingenieros de Google, Yahoo, Facebook y Oracle para comercializar Hadoop.
2012
Snowflake fundada por ex-arquitectos de Oracle. Apache Spark creado en AMPLab, UC Berkeley.
2013
Databricks fundada por los creadores de Apache Spark. Levanta Serie A liderada por a16z.
2015
Dremio fundada. Snowflake lanza disponibilidad general de su plataforma.
2019
Cloudera fusiona con Hortonworks creando la mayor empresa de datos open-source del mundo.
2020
Snowflake realiza el mayor IPO de software en la historia. Delta Lake (Databricks) se hace open-source.
2021–2022
Databricks lanza Unity Catalog. Dremio adopta Apache Iceberg como estándar. AWS lanza AWS Lake Formation.
2023–2024
Databricks adquiere MosaicML y Tabular. Snowflake lanza Arctic LLM. AWS libera S3 Tables con soporte Iceberg.

02 — Implementaciones empresariales

Casos Reales de Uso

Las grandes organizaciones han confiado en estas plataformas para resolver retos críticos de datos. A continuación, algunos de los casos más representativos.

Cloudera
Deutsche Telekom

Implementó CDP para unificar análisis de red en tiempo real e infraestructura on-premise, procesando petabytes de telemetría de 5G con cumplimiento GDPR.

Cloudera
Banco Santander

Usa Cloudera Data Platform para detección de fraude en tiempo real, procesando millones de transacciones diarias con modelos de ML integrados.

Databricks
Comcast

Procesa más de 100 billones de eventos diarios en su Lakehouse para personalización de contenido, reduciendo el tiempo de ingesta en un 60%.

Databricks
Shell

Implementó Databricks para análisis de sensores IoT en plataformas petroleras, optimizando el mantenimiento predictivo y reduciendo tiempos de inactividad.

AWS Big Data
Netflix

Utiliza Amazon EMR, S3 y Kinesis para procesar miles de millones de eventos de visualización por día, potenciando su motor de recomendaciones.

AWS Big Data
Airbnb

Amazon Redshift y AWS Glue forman el núcleo del pipeline de datos de Airbnb, procesando datos de millones de reservas en más de 190 países.

Snowflake
Pfizer

Durante la pandemia de COVID-19 usó Snowflake para compartir datos de ensayos clínicos en tiempo real entre investigadores de diferentes organizaciones.

Snowflake
JetBlue Airways

Migró su data warehouse a Snowflake, reduciendo costos de infraestructura en 40% y habilitando análisis de rutas y precios en tiempo real.

Dremio
FINRA

La autoridad reguladora de EE.UU. usa Dremio para analizar miles de millones de transacciones bursátiles diarias sin ETL, detectando irregularidades del mercado.

Dremio
Maersk

La naviera global usa Dremio para consultar datos de contenedores, rutas y logística directamente desde su data lake, sin replicar datos a un warehouse separado.
